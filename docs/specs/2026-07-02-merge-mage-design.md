## 1. 게임 디자인

### 코어 루프 (Idle Sword Master의 검증된 훅 이식)
- **소환 바닥 규칙**: 소환되는 마법서 레벨 = `max(1, 최고 마법서 레벨 − 8)`. 같은 레벨 2권 머지 → +1레벨. 최고 레벨 갱신 시 소환 바닥 동반 상승 (토스트 알림).
- **그리드**: 4행×5열. 최상단 행 6칸... → **장착 슬롯 6개** (별도 행, 그리드 위). 장착된 책들이 시전 인스턴스 (수량=화력). 드래그로 장착/해제.
- **소환**: 골드 비용 `cost = 20 × 1.22^(소환레벨)` (파라미터). Auto Buy / Auto Merge 토글 무료 제공.
- **원소 3종**: 화염(광역 스플래시)/냉기(슬로우)/신성(보스 +100% 데미지). 소환 시 랜덤. 머지는 같은 레벨이면 원소 무관, 결과는 두 재료 중 랜덤 원소.
- **전투**: 마법사 고정 위치(좌), 몬스터 웨이브 우측 접근. 10웨이브마다 보스 관문(실패 시 현 스테이지 반복). 스테이지별 몬스터 HP `= 15 × 1.28^stage`.
- **데미지**: 책 데미지 `= 5 × 1.35^lv × 슬롯강화배수 × 스킬배수 × 환생배수`.

### 메타 시스템
- **마법사 레벨/스킬**: 처치 EXP → 레벨업 → 스킬 포인트. 스킬 4종(소환 레벨 보너스/시전속도/골드 획득/치명타). **무료 리셋 언제든**.
- **환생(Arcane Rebirth)**: 스테이지 10+부터 가능. 마나 크리스탈 `= floor(stage^1.5 / 10)` 획득, 크리스탈당 전 데미지 +10%. 진행 초기화(스킬/크리스탈 유지).
- **오프라인 정산**: 서버 계산. `골드 = 최근 초당골드 × min(경과, 8h) × 0.6`. 클라 시계 무시, 서버 시각 기준.
- **리더보드**: 최고 스테이지 Top 100. 닉네임(자동생성, 수정가능).

## 2. 아키텍처

```
merge-mage/  (public GitHub repo — 커밋 히스토리가 루프 증거)
├─ src/                 # Vite 8.1 + TS
│  ├─ game/             # Phaser 4.2 씬 (전투 연출 전용)
│  ├─ ui/               # React 오버레이 (그리드·상점·스킬·환생·리더보드)
│  ├─ engine/           # 순수 TS 이코노미 엔진 (Phaser/React 무의존, 시드 RNG, 결정론)
│  │  └─ simulate.ts    # 밸런스 시뮬레이터 (n시간 플레이 곡선 검증)
│  └─ bridge/           # Phaser↔React 이벤트 버스
├─ api/                 # Vercel 서버리스 (Hono): save / offline-claim / leaderboard
├─ public/assets/       # 픽셀 에셋 (CREDITS.md에 출처·라이선스)
├─ LOOP.md              # 에이전트가 매 루프 1줄씩 기록 (심사위원이 첫번째로 읽음)
└─ .github/workflows/testsprite.yml  # push→배포→TestSprite 게이트 (+5점)
```
- 상태 미러링: 게임 핵심 상태를 DOM `data-*` 속성으로 노출 (TestSprite 브라우저 에이전트가 검증 가능하게).
- DB: Neon Postgres (saves, leaderboard). 익명 디바이스 토큰 인증.
